# Showroom

OMNIA-VALIDATION is the public showroom of the OMNIA ecosystem.

The goal is simple:

    show evidence before theory.

A visitor should not need to understand every repository before seeing value.

The repository should make one thing visible immediately:

    a structural claim can be tested by artifacts.

---

## Showroom path

The preferred public path is:

    clone
      -> install
      -> run tests
      -> inspect artifact
      -> read boundary
      -> reproduce or falsify

---

## What should be visible

The strongest showroom contains:

1. a minimal reproducible command;
2. a small validation dataset or example;
3. an output artifact;
4. a failure or fragility example;
5. a short report;
6. a boundary statement.

The visitor should be able to answer:

    What was tested?
    What changed?
    What remained stable?
    What collapsed?
    Where is the artifact?
    What does this prove?
    What does this not prove?

---

## Public demonstration logic

The best OMNIA-VALIDATION demonstration is not:

    OMNIA is right.

The best demonstration is:

    A surface-valid output can be structurally fragile.
    The fragility can be measured.
    The measurement can be inspected.
    The claim can be bounded.

---

## First-time visitor compression

    Surface correctness can lie.
    Structural stability is harder to fake.
    OMNIA-VALIDATION shows the evidence trail.

---

## Repository role

This repository should be optimized for reviewers, engineers, and external observers.

It should answer:

    Can I run it?
    Can I inspect the result?
    Can I reproduce it?
    Can I see where it fails?
    Can I verify the boundary?

If the answer is yes, the showroom works.
