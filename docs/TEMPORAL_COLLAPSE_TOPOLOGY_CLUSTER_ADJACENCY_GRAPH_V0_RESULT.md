## Threshold Transition Pattern

The threshold edges often move in the direction:

```text
higher cost -> lower cost
```

Example:

```text
cost=4|remaining=1
->
cost=3|remaining=1
```

```text
cost=3|remaining=1
->
cost=2|remaining=1
```

```text
cost=2|remaining=1
->
cost=1|remaining=1
```

```text
cost=1|remaining=1
->
cost=0|remaining=1
```

This reflects the rule:

```text
transition_cost = sum(max(0, run_length - (confirmation_window - 1)))
```

As confirmation window increases, reducible mass often decreases.

So threshold transitions tend to compress cost.

---

## Geometry Transition Pattern

Geometry edges often move in the direction:

```text
lower cost -> higher cost
```

Example:

```text
cost=0|remaining=1
->
cost=1|remaining=1
```

```text
cost=1|remaining=1
->
cost=2|remaining=1
```

```text
cost=2|remaining=1
->
cost=3|remaining=1
```

This reflects ordered geometry progression within families.

When run length or collapse mass increases, transition cost often increases.

So geometry transitions tend to expand cost.

## Self-Edges

The graph detected:

```text

```

Self-edges mean:


They occur when a threshold or geometry perturbation does not change the invariant cluster.

A self-edge means local stability under that perturbation type.
